every time user hits save from webapp, this dir first gets cleared, then everything in storage dir is copied here; and all contents are converted/dithered. there is room to mess with what parts have what jobs, e.g. moving converter to frontend, or converting upfront and not even storing original user uploads.
this dir is directly looked into for images to draw. i was thinking of making localhost get requests here to 'download' them and have a copy dir for display, but that's not necessary. (either way, the server is of course going to be off most of the time, only on when user is uploading or changing settings)
